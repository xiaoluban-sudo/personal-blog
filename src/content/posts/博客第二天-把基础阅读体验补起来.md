昨天我们把博客从 0 搭了起来：能写 Markdown，能生成文章页，也能部署到 GitHub Pages。今天的目标不是做一个复杂系统，而是把它从“能访问”推进到“更像一个可以长期使用的博客”。

这一版主要补的是阅读、发现和维护体验。

## 搜索

我先加了站内搜索页。

第一版搜索只覆盖标题、简介、分类和标签，后来又把正文内容也纳入索引。这样以后文章多起来后，不需要只靠分类和标签回忆文章在哪里，直接搜正文里的关键词也能找到。

搜索是纯前端实现，没有后端，也不需要数据库。构建时把公开文章整理成一个小索引，页面里用 JavaScript 实时筛选。

## 暗色模式

接着加了亮色 / 暗色模式切换。

这个功能看起来只是视觉偏好，但它会影响我愿不愿意晚上继续写东西。实现上用了 CSS 变量维护主题颜色，再用 `localStorage` 记住浏览器里的选择；如果没有手动选择，就跟随系统主题。

## 文章目录

文章页现在会根据 Markdown 里的二级、三级标题自动生成目录。

短文章不会强行展示目录，只有标题数量足够时才出现。这个细节挺重要：工具应该在需要的时候帮忙，不需要的时候安静待着。

## 分类、标签和归档

今天还补了几个浏览入口：

- 分类总览页
- 标签总览页
- 文章归档页
- 分类详情页
- 标签详情页

分类适合表达文章的大方向，比如“技术”“随记”；标签适合表达更细的主题，比如 `Astro`、`部署`、`博客`。

归档页则更像时间线。以后回头看这个博客怎么一点点长起来，会很有意思。

## 阅读细节

最后补了一些小功能：

- 阅读时间估算
- 字数统计
- 上一篇 / 下一篇
- 复制文章链接
- 返回顶部
- 自定义 404 页面
- RSS
- Sitemap
- robots.txt
- 基础 SEO meta

这些功能单独看都不大，但加在一起，会让博客更完整。它们不是“炫技功能”，更像是一个博客日常使用时应该顺手存在的东西。

## 今天的感受

自己写博客和套模板最大的区别，大概是每一个小功能都能留下痕迹。

我不只是拥有了一个博客页面，也拥有了它成长的过程。今天加搜索，明天也许会加评论；今天改样式，后面可能会重新设计整个阅读页。每一次迭代都可以变成一篇文章。

这样挺好。博客不是一次性做完的项目，而是一块可以慢慢打磨的地方。

## 下一步

后面我想继续做几件事：

- 给文章加更好的代码块样式
- 支持草稿预览
- 增加站点统计
- 优化移动端导航
- 写更多真实文章，而不是只写博客本身

先把这个版本记下来。今天的博客，比昨天更像一个真正可以使用的博客了。
